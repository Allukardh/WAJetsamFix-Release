# Release policy

GitHub Releases are displayed in descending order, with the newest release at the top.

- The latest stable Release is the only Release intended to contain a downloadable `.deb`.
- Older Releases remain separate and retain their curated changelogs, but do not retain downloadable packages.
- Release notes repeat the relevant changelog because many users read the Release page without opening the repository changelog.
- Public release notes must not expose source code, private implementation details, credentials, internal paths or raw development records.
- The public sponsor call-to-action is placed after the latest release changelog so it appears before the Assets section.
